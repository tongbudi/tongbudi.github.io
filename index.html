BitTorrent Sync 从这开始

Apr 26th, 2013

入门指南

  1. 在你希望同步的设备上安装 BitTorrent Sync。
  2. 在其中一个设备上选择要同步的文件夹并为这个文件夹生成一个密钥。密钥是随机和唯一的，它就像一把钥匙，用来在一个同步网络上连接多个设备。
  3. 在其他设备上，选择一个要同步的文件夹并输入相同的密钥。
  4. 在你希望同步的所有设备上保持 BitTorrent Sync 开启。
  5. 可以在任何时候通过程序接口在 Windows 和 Mac 上管理同步的文件夹，或者在 Linux 上通过 WebUI 和配置文件进行管理。

选择文件夹

BitTorrent Sync 允许同步任意数量的文件夹和文件。

要往 Sync 添加一个文件夹，在 Folders 标签页的底部，点击 ‘+’ 按钮（Mac）或 ‘Add’ 按钮（Windows）。然后在你的电脑上或外置驱动器上选择任意文件夹（要注意你必须有修改这个文件夹的权限）。

如果文件夹不存在，你可以在你的电脑上的任何地方创建一个新文件夹来进行上述操作。

生成/使用一个密钥

同步的文件夹通过一个普通密钥连接——每个要同步的文件夹分别有一个密钥。

要新增一个同步网络，点击 Generate 来创建一个新的密钥，这个密钥必须在每个希望同步的设备上输入。也可以通过输入其他人共享给你的密钥来加入一个同步网络。

如果需要找到密钥，可以通过在 Folders 标签页上右键点击文件夹来找回，可以将其复制到剪贴板或在 folder preferences 的 advanced 标签页查看。

单向同步

如果你希望只从你的电脑同步过来信息（而不要同步回去），可以通过使用只读密钥来实现。例如，你可以用只读密钥共享一个装满了照片和视频的文件夹给你的家人，这样除了你之外其他人就不能往这个文件夹中添加或删除文件。

只读密钥可以通过在 Folders 标签页上右键点击文件夹来找到，可以将其复制到剪贴板或在 folder preferences 的 Advanced 标签页查看。

在任意设备上使用只读密钥都可以完整同步，但对同步内容的修改将不会影响原始设备上的内容。你可以根据对不同设备的同步需求来使用只读密钥配合主密钥。使用只读密钥的设备无法对同步的文件夹生成新密钥。

单次密钥

出于安全考虑，你可能不会想要共享主密钥。这种情况下你可以创建一个只能使用一次且会在 24 小时内过期的密钥。可以是只读的也可以是全访问权限的密钥。单次密钥一旦被输入，设备将会收到一个只读的或全访问权限的永久密钥，以便在同步网络中使用。

程序首选项

可以在主窗口的 Preferences 标签页设置程序的首选项。

可以在这里修改用来在其他设备上显示的当前计算机名，配置更新和体型，修改监听的端口和使用 UPnP 映射。也可以对上传和下载进行限速。

这个标签页也可以访问到高级首选项。需要注意的是高级首选项是提供给专家用户使用的，普通用户如果要做修改的话风险自负。关于高级首选项的信息可以在 [用户指南] 找到。

文件夹首选项

除了可以修改整个程序的设置，BitTorrent Sync 还允许你为每个文件夹单独调整首选项。可以在 Folders 标签页通过双击或右键单击一个文件夹来进行设置。

通用选项标签页

BitTorrent Sync 通过预设的配置来确保最佳性能，当然你也可以根据自己的需要来进行调整：

  • 如果因为 NAT 问题而导致无法连接到其他设备时，使用中继服务器。
  • tracker server（追踪服务器）可以帮助在设备间通讯和互相发现。
  • “Delete Files to Sync trash（删除文件到同步回收站）”在所有设备中保存删除的文件到“.SyncTrash”文件夹。如果不选中该项，删除的文件将会被移动到系统的 Trash\Recycle Bin（不同的操作系统可能不一样）。
  • “Use predefined hosts（使用预定义的 hosts）”是用来为已知客户端指定 ip:port 或 host:port 的选项。这样，如果你的一个设备使用静态可访问的 IP，节点之间就可以直接连接。

高级选项标签页

你可以在高级文件夹首选项为同步文件夹复制、修改或生成新的密钥。

  • Secret（密钥）是一个用来连接文件夹的 32 位字符主键。所有使用这个密钥添加的文件夹，都将被授予双向同步的全部权限。你可以随时修改，或使用一个 40 个以上字符的 Base64 字符串来替换主密钥（文件夹密钥若有改动，就必须在同步网络中的其他设备上重新输入）。
  • Read only secret（只读密钥）是基于主密钥生成的，用来做单向同步。任何使用只读密钥的文件夹都可以完全同步，但对文件夹的修改将不会作用到原始设备上。而被授予只读密钥的文件夹，自然也无法修改它的密钥。
  • One-time secret（单次密钥）是一个由主密钥操作的（区别于只读密钥）文件夹安全选项。可以生成完全访问或只读短键（16 个字符），它智能使用一次，并且必须在 24 小时内被激活。连接一旦被建立，通过单次密钥添加的文件夹，将会收到一个生成单次密钥设备生成的，持久并且是只读的主键。

Linux 和 NAS

WebUI

WebUI 是访问 BitTorrent Sync 的默认方法，可以通过 YourLocalIP:8888/gui 来访问。

和桌面程序一样，可以通过 WebUI 来管理通用设置和文件夹首选项。

配置文件

通过配置文件提供了稍多一点的设置选项，它是一个 JSON 格式的文件。

通过配置文件设置首选项：

  1. 通过执行 –dump-sample-config 来获取示例配置，可以看到选项和注释的列表。
  2. 可以保存配置文件到任意地方。例如，要保存到当前文件夹，运行 ./btsync –dump-sample-config > sync.conf
  3. 定制配置项，并使用修改后的配置文件来运行 BitTorrent Sync。例如，如果配置文件和 BitTorrent Sync 在同一个文件夹，可以运行：./btsync –config sync.conf

在通过修改过的配置文件来运行 BitTorrent Sync 之前，确保进行 JSON 有效测试。否则，你可能会得到错误消息。

BitTorrent Sync 提供了大量的配置选项来确保你能得到你想要的。默认的 Sync（同步）设置基于 BitTorrent 广泛的普通网络和 P2P 网络经验，适用于大多数的用户。

BitTorrent Sync，终极同步解决方案？

Apr 25th, 2013

如果 BitTorrent 开始做云端同步服务，你还会用别家的吗？

这并不仅仅是“如果”——今天 CNet 向我们介绍了 BitTorrent 推出的一款新服务 BitTorrent Sync，它帮助用户在多台设备之间同步个人文件。

BitTorrent Sync 采用点对点技术在不同设备之间同步文件。据 BitTorrent 官方发言人表示，BitTorrent Sync 将支持设备之间的直接同步，而不需要在 Mac OS、Windows 或 Linux 系统中安装其他任何本地软件，也不需要使用 NAS 技术进行云缓存。另外，文件在同步过程中将自动加密。

此前，BitTorrent 在官方博客中表示：“我们希望能够把所有东西放在一起，于是我们就想要开发一款全新的分布式同步产品，协助在多台计算机之间管理个人文件。”

BitTorrent Sync 的操作方式非常简单：

成功安装软件后，用户可以设置一到两个同步文件夹，然后就会收到一串 21 字节的验证码，将这串验证码复制到用户想要同步的设备上去，这样就可以将这些设备联接起来了。

这款工具的最大优势在于，共享过程中不需要任何第三方的同步介质，而同步文件的最大存储量也只会由你的硬盘来决定。

据悉，这款工具目前还不具备版本控制和备份管理功能，不过用户可以选择一些补充手段，比如和 Time Machine 等备份软件一起使用。

BitTorrent Sync 是在今年 1 月正式被公布的，目前仍处于 Alpha 测试的初期，而 BitTorrent 仍在寻找自愿测试这款服务的志愿者。

BitTorrent 在官方博客中表示：“我们希望用户能帮助我们，大家一起开发这款出色的产品。如果你愿意使用一款还处在开发阶段的软件，和我们一起寻找更好的同步方式，那么我们十分愿意倾听你的意见。”
